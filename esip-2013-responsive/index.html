Field Catalog 2 Goals

DRY: Don't Repeat Yourself

  • avoid writing new interface for every device form factor

    • redundant
    • more work to support

  • How to achieve this in a web interface?

use contemporary technologies and techniques

    • don't reinvent the wheel
    • lots of smart and talented folks have done this before and well

Responsive Design

Responsive web design is a web design approach aimed at crafting sites to provide an optimal viewing experience— easy reading and navigation with a minimum of resizing, panning, and scrolling— across a wide range of devices (from desktop computer monitors to mobile phones).

http://en.wikipedia.org/wiki/Responsive_web_design

When responsive isn't enough:

Viewport

  • By default, mobile browsers try to mimic desktop display
  • this results in pages scaled down to fix full web page onto a small mobile display
  • Apple implemented hooks to a viewport meta tag
  • Allows assignment of page width, based on pixels, device width, scaling, etc.

User-Agent inspection

  def mobile_device?
    request.user_agent =~ /Mobile|webOS|Android/i
  end

  if mobile_device?
    # mobile-specific stuff
  end
  • allow user to select other interface, store interface selection as cookie
  • not immune to User-Agent spoofing

Lessons Learned

  • offload the heavy-lifting to a robust, supported framework, e.g.
    • Bootstrap
    • Foundation
  • it's easy to override styles so that application does not look like Just Another Bootstrap App
    • you may or may not want to do this
  • no silver bullet
  • keep it simple
  • progressively enhance
  • use emulators and simulators when available
  • test on anticipated hardware
